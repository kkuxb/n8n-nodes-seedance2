# Milestone v1.2: Seedream Image Generation, UX Iteration, and Audit Reconciliation

**Status:** ✅ SHIPPED 2026-04-20
**Phases:** 08-13
**Total Plans:** 11

## Overview

v1.2 delivered Seedream 5.0 lite image generation inside the existing `Seedance` node while preserving the shipped Seedance video lifecycle. The milestone covered image-generation contract design, request shaping and validation, binary image output, partial-failure handling, mode-first UX refinement, user-facing documentation, and a final audit-reconciliation phase to align requirements and verification artifacts with shipped behavior.

## Milestone Summary

**Key Decisions:**

- Keep image generation inside the existing `Seedance` node instead of creating a separate node.
- Reuse `SeedanceApi` API Key credentials for both Seedance video and Seedream image generation.
- Public image reference inputs are URL and binary; base64/data URL remains internal compatibility fallback.
- Default image output is n8n binary, not URL-only.
- Preserve shipped video lifecycle behavior as a hard non-regression rule.

**Issues Resolved:**

- Added full Seedream image generation support under the existing node contract.
- Closed prior milestone audit blockers by reconciling requirements and backfilling missing phase verifications.
- Locked execute-level group-image `max_images` request shaping with regression coverage.

**Issues Deferred:**

- Historical Phase 12 verification wording still reflects a pre-watermark-exposure state.
- Internal compatibility fallback fields remain in image reference handling even though the public UI exposes only URL and binary.

**Technical Debt Incurred:**

- PNG branding remains accepted lint-related technical debt until the project intentionally switches back to SVG or changes validation policy.
