## Critical Gaps

### IMG-04: Base64 Reference Input Is Not Reachable From UI

The milestone requirement says users can pass a single reference image via URL, base64, or binary.

Evidence:

- Execute path still supports `referenceImageSource === 'base64'` in `nodes/Seedance/Seedance.node.ts`.
- `referenceImageBase64` still exists in `nodes/Seedance/description/image.operation.ts`.
- `referenceImageSource` options in `nodes/Seedance/description/image.operation.ts` expose only `url` and `binary`.

Impact:

- A normal n8n user cannot select base64 in the image-to-image UI.
- Lower layers support the capability, but the user-facing flow is broken.

Resolution options:

- Restore `Base64 / Data URL` as a public `referenceImageSource` option, or
- Update `REQUIREMENTS.md` and acceptance scope to remove base64 from v1.2 public UI, then re-verify the changed requirement contract.

### Phase 08-11 Missing Formal Verification Files

The audit workflow requires each completed phase to have `*-VERIFICATION.md`.

Missing files:

- `.planning/phases/08-seedream-image-operation-contract-ux-skeleton/08-VERIFICATION.md`
- `.planning/phases/09-image-payload-builder-input-normalization-validation/09-VERIFICATION.md`
- `.planning/phases/10-image-execution-path-binary-output-partial-failure-mapping/10-VERIFICATION.md`
- `.planning/phases/11-regression-coverage-documentation-release-hardening/11-VERIFICATION.md`

Impact:

- Even if summary artifacts show tests passed, the milestone does not satisfy the audit gate for phase-level verification.

Resolution:

- Run `/gsd-verify-work` or equivalent retrospective verification for Phases 08-11, or
- Create a gap-closure phase that generates and reviews those missing verification artifacts.

## Tech Debt

### VAL-IMG-02 Requirement Status Drift

`REQUIREMENTS.md` still shows `VAL-IMG-02` unchecked, but implementation evidence exists:

- `nodes/Seedance/shared/validators/seedreamImage.ts` validates resolution/aspect-ratio through recommended-size mapping.
- `test/seedreamImageValidation.test.ts` covers invalid resolution/aspect ratio.

This should be synchronized during gap closure.

### Image Watermark Scope Drift

`REQUIREMENTS.md` says image watermark is out of scope, but post-Phase 12 debug work intentionally exposed `imageWatermark` default-off because the API default adds a watermark.

Evidence:

- `nodes/Seedance/description/image.operation.ts` exposes image watermark.
- `nodes/Seedance/shared/mappers/seedreamImagePayload.ts` sends `watermark`.
- `README.md` documents image watermark default-off.

This is likely an acceptable product correction, but the planning contract needs updating.

### Group Image Execute-Level Assertion

There is mapper/validator coverage for group image options and execute coverage for multi-image outputs, but a direct execute-level assertion for `sequential_image_generation_options.max_images` would make the end-to-end contract stronger.
