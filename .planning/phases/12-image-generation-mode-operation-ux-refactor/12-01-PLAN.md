## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| n8n UI parameters → node execution | User-controlled mode, operation, prompt, option, and reference-image field values determine which execute branch runs. |
| Saved workflows → refactored parameter schema | Existing workflow JSON may contain old parameter values such as `operation=generateImage`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | T | `displayOptions` in description files | mitigate | Gate every video field with `generationMode=['video']` and every image field with `generationMode=['image']` so unrelated fields cannot be accidentally submitted from the visible UI. |
| T-12-02 | I | `Seedance.node.ts` subtitle/operation selectors | mitigate | Remove image generation from the video operation list and cover the selector contract with `seedreamImageOperationContract.test.ts`. |
| T-12-03 | D | Saved workflow compatibility | accept | Phase 12 intentionally changes the image UX before public v1.2 release; Plan 12-02 still handles image execution by generation mode and can optionally accept `operation=generateImage` as an internal compatibility fallback if tests require it. |
</threat_model>
