<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| n8n input parameters → Seedream payload | Prompt, option toggles, URL strings, base64 strings, and binary property names are user-controlled and become API payload fields. |
| n8n binary data → in-memory base64 conversion | User-selected binary properties are read into memory and converted to data URLs before validation and API submission. |
| Refactored dispatch → existing video branches | New `generationMode` selection must not route video operations into image code or vice versa. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04 | T | `collectSeedreamReferenceImages()` | mitigate | Trim comma-separated URL/binary entries, drop empty entries, and pass all references through existing `validateSeedreamImageInput()` count/MIME/size rules. |
| T-12-05 | D | binary reference processing | mitigate | Continue using `this.helpers.assertBinaryData()` and existing 10MB validator before API submission; do not add background downloads or new dependencies. |
| T-12-06 | E | mode-aware execute dispatch | mitigate | Default absent `generationMode` to video and keep video branches keyed by existing operation values; cover with `seedanceVideoRegression.test.ts`. |
| T-12-07 | I | image output JSON | mitigate | Continue using `mapSeedreamImageResponse()` so raw base64 is attached as binary and not leaked into JSON. |
</threat_model>
